Issue:
I’m using Scala 3.7.3 in my project, but the Akka dependencies in the build file are set to version 2.10.0, which doesn’t seem to exist in Maven repositories.
Also, I’m getting resolution errors for akka-actor and akka-http artifacts compiled for Scala 3.

Solution
Here is the clean setup where your Akka repository token lives in a separate sbt file, and your app compiles with Scala 3.7 and Akka 2.10.
Create a normal sbt project layout
project/build.properties
sbt.version=1.10.7
build.sbt
ThisBuild / scalaVersion := "3.7.0"
ThisBuild / organization := "com.example"
ThisBuild / version := "0.1.0-SNAPSHOT"
lazy val akkaVersion = "2.10.17"
lazy val root = project
  .in(file("."))
  .settings(
    name := "akka210-scala37-demo",
    libraryDependencies ++= Seq(
      "com.typesafe.akka" %% "akka-actor-typed" % akkaVersion,
      "com.typesafe.akka" %% "akka-stream" % akkaVersion,
      "com.typesafe.akka" %% "akka-slf4j" % akkaVersion,
      "com.typesafe.akka" %% "akka-actor-testkit-typed" % akkaVersion % Test,
      "ch.qos.logback" % "logback-classic" % "1.5.16" % Runtime
    )
  )
Put the Akka secure repo resolver in a separate sbt file
Akka artifacts are served from a secure, tokenized repository, so sbt must add a resolver that includes your token. (Akka Documentation)
Option A, recommended: keep it out of your repo using a global sbt file
Create this file on your machine
~/.sbt/1.0/akka-commercial.sbt
ThisBuild / resolvers += {
  val token = sys.env.getOrElse(
    "AKKA_REPO_TOKEN",
    sys.error("Missing AKKA_REPO_TOKEN environment variable")
  )
  "Akka library repository".at(s"https://repo.akka.io/$token/secure")
}
This pattern is explicitly used in Akka docs as a global sbt file. (Akka Documentation)
Option B, project local: separate file next to build.sbt
Create a file in the project root, for example
akka-commercial.sbt
ThisBuild / resolvers += {
  val token = sys.env.getOrElse(
    "AKKA_REPO_TOKEN",
    sys.error("Missing AKKA_REPO_TOKEN environment variable")
  )
  "Akka library repository".at(s"https://repo.akka.io/$token/secure")
}
sbt automatically loads all .sbt files in the project root, so this stays separate without extra wiring.
Set your token once in the shell
Get a token from the Akka account page. (account.akka.io)
macOS or Linux
export AKKA_REPO_TOKEN="paste_your_token_here"
Compile
sbt compile
If the token or resolver is missing, dependency resolution will fail because Akka 2.10 is not in the public Maven repos, it is behind the secure repository. (Akka Documentation)
